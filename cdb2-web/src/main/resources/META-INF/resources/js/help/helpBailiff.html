Purpose

This page allows you to manage the bailiffs in your National Bailiff Directory.

How it works ?

Page structure

The upper part of this page contains a set of criteria to filter out the list of bailiffs and ease the research of a specific bailiff. Simply enter a value in the filter fields and the list is automatically updated with the bailiffs that match this criteria. The match is done on any part of the corresponding value of the Bailiff.

The lower part of the page contains the list of bailiffs. The two arrows next the title at the top of the list are used to sort the list of bailiffs with this column values. On top of the list, you can choose the size to display per page. If the number of bailiffs is greater to the page size, there is a pagination mechanism that appears on top of the list. You can navigate through the pages by selecting a specific page or by using the arrows to go to the previous/next page.

Button 'download template'

This download an Excel template file. This template can be used to initialize the bailiff directory from an external existing source.

Button 'Import'

This button accepts an Excel file based on the template to import bailiffs in a single operation. Only the fields available in the template are imported. The bailiff definition can be afterwards completed by editing them one by one.

Button 'Export'

This button exports the list of bailiffs in an Excel file. The format of this Excel file has the same structure as the template file.

Button 'Add a bailiff'

This button opens a window to enter a new bailiff. For a new bailiff, you can enter the following elements :

  • National identifier : it is commonly a national identifier assigned to a bailiff. There is no format validation.
  • Name : the name or the denomination of the bailiff. This is a mandatory field.
  • Address 1 : the street name and number. This is a mandatory field.
  • Address 2 : an optional line to enter the address.
  • City/Postal Code : the ZIP code and the city of the address. This is a mandatory field.
  • Phone : This is an optional field.
  • Mail : the mail adress of the bailiff. This is an optional field.
  • Website : The website of the bailiff. This is an optional field.
  • Accepted languages : The list of accepted languages by the bailiff.
  • Comments : a free text to annotate the bailiff. This is an optional field.
  • Instruments : the list of legal instruments for which this bailiff is competent. At least one instrument is required.
  • GeoAreas : the name of the GeoArea on which the bailiff is competent. The GeoArea is a list of ZIP codes that defines an geographical area. If this notion is not relevant for a country, a global geoarea can be defined to cover the whole country.

Button 'Edit a bailiff'

This button opens a window to edit the bailiff information. This is the same window as described in the 'add a bailiff' function.

Button 'Delete a bailiff'

This button marks a bailiff as deleted. The data are not lost but the bailiff is considered as no more active.
